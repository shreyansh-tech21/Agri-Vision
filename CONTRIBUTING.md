# Contributing to  🤝 Agri-Vision

Thank you for your interest in contributing to Agri-Vision 🚀. 
We welcome contributions including
-🐛Bug fixes 
-✨ New features
-🎨UI improvements
-📖Documentation updates
-🧪Suggestions.

Your contributions help improve Agri-Vision for developers, researchers, and farmers.

## 📌 Before You Start

- Fork the repository
- Read the README carefully
- Check existing issues before creating a new one
- Follow the project structure and coding style
- Make sure your changes are relevant and focused

## 🌱 Beginner Contribution Workflow

1. Fork the repository
2. Clone your fork locally
3. Create a new branch
4. Make your changes
5. Commit your changes
6. Push changes to GitHub
7. Open a Pull Request

## Getting Started

### 1. Fork the Repository

Click on the Fork button at the top right corner of this repository.

### 2. Clone Your Fork

Clone the forked repository to your local machine.

```bash
git clone https://github.com/your-username/Agri-Vision.git
cd Agri-Vision
```

### 3. Navigate to the Project Directory
Move into the project directory.

```bash
cd agri-vision
```

### 4. Create a New Branch

Create a separate branch for your contribution.

```bash
git checkout -b improve-contributing-guide
```
## 5. Sync Your Fork with Upstream Repository

Keep your fork updated with the latest changes from the original repository:

```bash
git remote add upstream https://github.com/neeru24/Agri-Vision.git
git pull upstream main
```
## 6. Run Project Locally

⚙️
Install dependencies and start the project locally:

```bash
pip install -r requirements.txt
python app.py
```
## 7. Run Tests

Before submitting your contribution, run the test suite to ensure everything works properly:

```bash
python -m pytest
```

##  ✨Making Contributions

You can contribute in the following ways:

-🐛 Fix bugs
-🎨 Improve UI/UX
-✨ Add new features
-📖 Improve documentation
-⚡ Optimize performance
-🧪 Add tests

## Commit Guidelines

Write clear and meaningful commit messages describing your changes.

## Push Changes

Push your branch to your forked repository.

```bash
git add .
git commit -m "Improve CONTRIBUTING guide for beginners"
```



## Create a Pull Request

- Open your forked repository
- Click on Compare & Pull Request
- Add a proper title and description
- Submit the PR

## Pull Request Rules

- PR should solve only one issue
- Avoid unnecessary changes
- Make sure the project runs properly
- Follow clean coding practices
- Keep PR descriptions detailed
- Test changes before submitting

## Reporting Issues

When creating an issue, include:

- Proper title
- Detailed description
- Screenshots if applicable
- Steps to reproduce the bug

## Code Style

- Use meaningful variable names
- Keep code clean and readable
- Add comments where necessary

## Code of Conduct

Please be respectful and maintain a positive environment for everyone.

## ❓ Need Help?

If you get stuck during contribution:

- Open an issue
- Ask maintainers politely
- Check existing discussions and pull requests
